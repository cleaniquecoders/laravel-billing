# iPay88

**Shape:** Form-POST · **Native recurring:** no (one-time per payment) · **SDK:** none — you
build a signed form and POST it to iPay88's entry URL.

iPay88 has no subscription concept: each payment is a one-off redirect. The customer's browser
must **POST** signed fields to `entry.asp`, so this recipe uses the package's
[`RedirectForm`](../../src/Gateways/Support/RedirectForm.php) bridge — `createCheckout()` returns
the `billing.gateway.redirect` URL, which renders an auto-submitting form. Signatures use
[`SignsPayloads::sha1Base64`](../../src/Gateways/Concerns/SignsPayloads.php).

- Reference: <https://github.com/karyamedia/ipay88> ·
  <https://github.com/cchitsiang/ipay88/blob/master/IPay88.class.php>

## Signature (verified against the reference)

`base64(SHA1(MerchantKey . MerchantCode . RefNo . Amount . Currency))`, where **`Amount` has all
commas and dots stripped** (`"1,499.00"` → `"149900"`). The response signature appends `PaymentId`
before `RefNo` and `Status` at the end.

## Driver

Paste into `app/Billing/Ipay88Gateway.php`:

```php
<?php

namespace App\Billing;

use CleaniqueCoders\LaravelBilling\Contracts\Billable;
use CleaniqueCoders\LaravelBilling\Contracts\PaymentGateway;
use CleaniqueCoders\LaravelBilling\DataTransferObjects\CheckoutIntent;
use CleaniqueCoders\LaravelBilling\DataTransferObjects\WebhookEvent;
use CleaniqueCoders\LaravelBilling\Enums\PlanInterval;
use CleaniqueCoders\LaravelBilling\Enums\WebhookEventType;
use CleaniqueCoders\LaravelBilling\Gateways\Concerns\SignsPayloads;
use CleaniqueCoders\LaravelBilling\Gateways\Support\RedirectForm;
use CleaniqueCoders\LaravelBilling\Models\Plan;
use CleaniqueCoders\LaravelBilling\Models\Subscription;
use Illuminate\Http\Request;
use Illuminate\Support\Str;

class Ipay88Gateway implements PaymentGateway
{
    use SignsPayloads;

    private const ENTRY_URL = 'https://www.mobile88.com/epayment/entry.asp';

    /** @param array{merchant_code:string,merchant_key:string,payment_id?:string} $config */
    public function __construct(private array $config) {}

    public function createCheckout(Billable $billable, Plan $plan, PlanInterval $interval, string $returnUrl): CheckoutIntent
    {
        $refNo = 'SUB'.Str::upper(Str::random(12));
        $amount = number_format($plan->priceCents($interval) / 100, 2, '.', '');   // "49.00"
        $currency = (string) config('billing.currency', 'MYR');

        $fields = [
            'MerchantCode' => $this->config['merchant_code'],
            'PaymentId' => $this->config['payment_id'] ?? '',
            'RefNo' => $refNo,
            'Amount' => $amount,
            'Currency' => $currency,
            'ProdDesc' => $plan->name.' ('.$interval->value.')',
            'UserName' => $billable->billingName(),
            'UserEmail' => $billable->billingEmail(),
            'UserContact' => '',
            'Remark' => '',
            'Lang' => 'UTF-8',
            'ResponseURL' => $returnUrl,
            'BackendURL' => route('billing.webhook', ['gateway' => 'ipay88']),
            'Signature' => $this->requestSignature($refNo, $amount, $currency),
        ];

        $token = RedirectForm::sign(self::ENTRY_URL, $fields);

        return new CheckoutIntent(route('billing.gateway.redirect', ['token' => $token]), $refNo);
    }

    public function cancel(Subscription $subscription): void
    {
        // One-time gateway — nothing to cancel upstream.
    }

    public function parseWebhook(Request $request): ?WebhookEvent
    {
        $refNo = (string) $request->input('RefNo');
        $amount = (string) $request->input('Amount');
        $currency = (string) $request->input('Currency');
        $status = (string) $request->input('Status');
        $paymentId = (string) $request->input('PaymentId');

        $expected = $this->responseSignature($paymentId, $refNo, $amount, $currency, $status);

        if (! $this->signaturesMatch($expected, (string) $request->input('Signature'))) {
            return null;
        }

        $providerEventId = 'ipay88-'.$request->input('TransId', $refNo);

        if ($status !== '1') {
            return new WebhookEvent(
                type: WebhookEventType::PaymentFailed,
                externalId: $refNo,
                providerEventId: $providerEventId,
                rawPayload: $request->all(),
            );
        }

        return new WebhookEvent(
            type: WebhookEventType::SubscriptionActivated,
            externalId: $refNo,
            amountCents: (int) round(((float) $amount) * 100),
            providerEventId: $providerEventId,
            rawPayload: $request->all(),
        );
    }

    private function requestSignature(string $refNo, string $amount, string $currency): string
    {
        return $this->sha1Base64(
            $this->config['merchant_key'].$this->config['merchant_code']
            .$refNo.$this->stripAmount($amount).$currency
        );
    }

    private function responseSignature(string $paymentId, string $refNo, string $amount, string $currency, string $status): string
    {
        return $this->sha1Base64(
            $this->config['merchant_key'].$this->config['merchant_code']
            .$paymentId.$refNo.$this->stripAmount($amount).$currency.$status
        );
    }

    /** iPay88 strips commas and dots from Amount before signing. */
    private function stripAmount(string $amount): string
    {
        return str_replace([',', '.'], '', $amount);
    }
}
```

## Config

```php
// config/billing.php
'gateways' => [
    'ipay88' => [
        'driver' => App\Billing\Ipay88Gateway::class,
        'merchant_code' => env('IPAY88_MERCHANT_CODE'),
        'merchant_key' => env('IPAY88_MERCHANT_KEY'),
        'payment_id' => env('IPAY88_PAYMENT_ID', ''), // '' or 0 = show all methods
    ],
],
```

```dotenv
IPAY88_MERCHANT_CODE=...
IPAY88_MERCHANT_KEY=...
IPAY88_PAYMENT_ID=
```

If you resolve config inside the driver instead, register it: `Billing::extend('ipay88', fn () =>
new App\Billing\Ipay88Gateway(config('billing.gateways.ipay88')));`

## Webhook route

iPay88 posts the result to the **BackendURL** (set above to `billing.webhook` with `gateway=ipay88`).
Use the shared route from the [index](README.md#the-webhook-route-your-app-owns-it); the customer is
separately redirected to `ResponseURL` (your `billing.success` page).

## Sandbox

Request a sandbox **MerchantCode** + **MerchantKey** from iPay88. The `BackendURL` must be publicly
reachable (use a tunnel such as `ngrok`/Expose for local testing). Subscribe → you're auto-posted to
iPay88 → pay → iPay88 calls your BackendURL with `Status=1` → `WebhookProcessor` activates the
subscription and issues an invoice. A tampered redirect token 404s at `billing.gateway.redirect`.

## Renewals

One-time only — **re-checkout** to renew. Email the customer a link that re-runs
`Billing::checkout()` near `current_period_end`, or expire access on lapse.
