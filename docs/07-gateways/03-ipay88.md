# iPay88

**Driver:** `CleaniqueCoders\LaravelBilling\Gateways\Ipay88Gateway` (ships with the package) ·
**Shape:** Form-POST · **Native recurring:** no (one-time per payment) · **Extra deps:** none.

iPay88 has no subscription concept — each payment is a one-off redirect, and the browser must
**POST** signed fields to `entry.asp`. The driver returns the package's
[`billing.gateway.redirect`](../../routes/gateway.php) bridge URL, which renders an auto-submitting
form. Signatures use `base64(SHA1(...))`.

- Reference: <https://github.com/karyamedia/ipay88> ·
  <https://github.com/cchitsiang/ipay88/blob/master/IPay88.class.php>

## Enable

```php
// config/billing.php
'gateways' => [
    'ipay88' => [
        'driver' => CleaniqueCoders\LaravelBilling\Gateways\Ipay88Gateway::class,
        'merchant_code' => env('IPAY88_MERCHANT_CODE'),
        'merchant_key' => env('IPAY88_MERCHANT_KEY'),
        'payment_id' => env('IPAY88_PAYMENT_ID', ''),       // '' / 0 = all methods
        'backend_url' => env('IPAY88_BACKEND_URL'),          // your public webhook URL
    ],
],
```

```dotenv
IPAY88_MERCHANT_CODE=...
IPAY88_MERCHANT_KEY=...
IPAY88_PAYMENT_ID=
IPAY88_BACKEND_URL=https://your-app.test/webhooks/ipay88
```

Point `IPAY88_BACKEND_URL` at your webhook route (the shared
[`billing.webhook`](README.md#the-webhook-route-your-app-owns-it) with `gateway=ipay88`). iPay88
posts the result there; the customer is separately redirected to your `billing.success` page.

## Signature (verified)

`base64(SHA1(MerchantKey . MerchantCode . RefNo . Amount . Currency))`, where **`Amount` has commas
and dots stripped** (`"1,499.00"` → `"149900"`). The response signature inserts `PaymentId` before
`RefNo` and appends `Status`. `Status == 1` → activation, else `PaymentFailed`.

## Sandbox

Request a sandbox **MerchantCode** + **MerchantKey** from iPay88. The `backend_url` must be publicly
reachable (use a tunnel locally). Subscribe → auto-posted to iPay88 → pay → iPay88 calls your
backend with `Status=1` → `WebhookProcessor` activates and issues the invoice. A tampered redirect
token 404s at `billing.gateway.redirect`.

## Renewals

One-time only — **re-checkout** to renew (re-run `Billing::checkout()` near `current_period_end`).
